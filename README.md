## 📌 Описание задания

Целью работы является изучение менеджера пакетов **Composer**, принципов объектно-ориентированного программирования (ООП) на PHP и интеграция внешних данных через **публичные API**.

**Выполненный функционал:**

* Настройка инфраструктуры через Composer и установка библиотеки `Guzzle`.
* Создание класса `ApiClient` для выполнения HTTP-запросов.
* Интеграция API погоды (**wttr.in**) для отображения условий во время тренировки.
* Создание класса `UserInfo` для сбора технической информации о пользователе (IP, Browser).
* Работа с сессиями и Cookies для отслеживания последнего визита.
* **Штрафное задание:** Реализовано кэширование ответов API в JSON-файл (на 5 минут), обработка ошибок и динамическое обновление данных через `fetch`.

---

## ⚙️ Как запустить проект

1. **Клонировать репозиторий:**
```bash
git clone <ссылка на репозиторий>
cd nginx-lab

```


2. **Установить зависимости (внутри контейнера или локально):**
```bash
composer install

```


3. **Запустить контейнеры через Docker:**

```bash
   docker-compose up -d --build

```

4. **Открыть проект в браузере:**

```http
   http://localhost:8080

```

## 📂 Структура проекта

* `www/ApiClient.php` — класс для работы с API через Guzzle.
* `www/UserInfo.php` — класс для получения данных о пользователе.
* `www/process.php` — обработчик формы, логика кэширования и работы с Cookies.
* `www/index.php` — главная страница с выводом погоды и технической информации.
* `www/api_cache.json` — файл кэша API (создается автоматически).
* `composer.json` — список зависимостей проекта.

---

## 🧠 Реализация штрафного задания

1. **Кэширование:** Система проверяет время изменения файла `api_cache.json`. Если прошло менее 300 секунд, данные берутся из файла без запроса к API.
2. **AJAX Обновление:** На главной странице добавлена кнопка "Обновить погоду", которая отправляет фоновый запрос через `fetch` и обновляет данные без перезагрузки страницы вручную.
3. **Обработка ошибок:** Если API недоступно или вернуло некорректный ответ, пользователю выводится дружелюбное уведомление вместо системной ошибки.
